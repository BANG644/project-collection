# 🔬 MadsLorentzen/ai-job-search - 全方位深度调研

## 📌 一句话定位

AI 驱动的全流程求职自动化框架——基于 Claude Code 构建的 Agent 工作流，从岗位爬取、匹配评估、到 LaTeX 简历/求职信定制化生成，全程让 AI 代理帮你完成，物理学家出品。

## ⭐ 项目亮点

1. **非程序员的 Agent 工程力作**：作者 Mads Lorentzen 是地球物理学家（博士），不是软件工程师，用 Claude Code 构建了一个多 Agent 求职工作流——证明了"AI Agent 降低了软件开发门槛"这一命题本身
2. **Drafter-Reviewer 双 Agent 质检架构**：简历和求职信生成后，由第二个 Agent 扮演"批评者"角色进行审查修改，形成 AI 内容的质量闭环——这比"生成即交付"的思路高一个层次
3. **LaTeX 原生集成**：不依赖 Word/PDF 模板，直接生成并编译 LaTeX 简历和求职信，精度和排版可控性远超通用文档生成方案
4. **语言/地区无关的核心架构**：核心工作流（自我画像、匹配评估、应用生成）与语言/地区解耦，仅招聘平台搜索组件需要替换——架构设计体现了良好的关注点分离

## 🏗️ 项目架构全景

### 目录结构与设计哲学

```
ai-job-search/
├── .agents/                    # Agent Skills 定义
│   └── skills/
│       ├── jobbank-search/cli/  # 丹麦招聘平台 CLI 搜索工具 (Bun/TypeScript)
│       ├── jobindex-search/cli/ # Jobindex 搜索
│       ├── jobnet-search/cli/   # Jobnet 搜索
│       ├── linkedin-search/cli/ # LinkedIn 搜索
│       └── ...
├── CLAUDE.md                   # AI Agent OpenAPI 规范（核心交互定义）
├── cv/                         # LaTeX 简历模板 (moderncv)
├── cover/                      # LaTeX 求职信模板
├── documents/                  # 个人资料文件夹（CV PDF/证书/推荐信等）
└── .gitignore
```

**设计哲学**：
- **"CLAUDE.md 即 API"**：整个框架通过 CLAUDE.md 向 Claude Code 暴露一组命令行入口（`/setup`, `/scrape`, `/apply`, `/rank`），Agent 通过这个文件理解自己的工作空间
- **"Fork 即用"**：用户 Fork 后只需运行 `/setup` 填写一次个人资料，剩下的全部自动化

### 技术栈

| 层 | 技术 | 用途 |
|----|------|------|
| Agent 运行时 | Claude Code | 核心推理与 Agent 编排 |
| CLI 搜索工具 | Bun/TypeScript | 各招聘平台搜索适配器 |
| 文档模板 | LaTeX (moderncv) | 简历与求职信排版 |
| 简历编译 | lualatex / xelatex | PDF 编译 |
| 封面模板 | cover.cls (LaTeX) | 求职信模板 |
| 审计工具 | pdftotext (poppler) | ATS 关键词可解析性检查 |

### 核心配置一览

框架的核心"配置"不是 JSON/YAML，而是 **CLAUDE.md 中的人岗画像模板**——一个包含身份、教育、技能、偏好、底线等 15 个维度的结构化模板。这是框架最聪明的地方：**用 AI 代理的 OpenAPI（CLAUDE.md）替代传统配置文件**。

## 💡 应用场景与启发

### 典型使用场景

1. **批量精准求职**：每天运行 `/scrape` 拉取匹配岗位，用 `/apply` 一次性生成定制化申请，HR 反馈率显著高于海投
2. **跨国求职适配**：核心工作流不绑定具体市场，替换搜索适配器即可用于不同国家
3. **模拟面试准备**：利用个人画像 + 岗位描述，让 Claude 模拟面试官进行针对性的 Q&A 练习

### 可借鉴的解决方案模式

1. **Drafter-Reviewer 双 Agent 模式**（最有价值的可复用模式）：
   ```
   Agent A 撰写初稿 → Agent B 审查质疑 → Agent A 修订 → 最终交付
   ```
   这个模式可以应用于任何需要质量控制的内容生成场景：PR 描述撰写、技术文档生成、代码评审等。

2. **CLAUDE.md 即 API 接口层**：
   不写 yargs/commander 等传统 CLI 框架，而是通过 CLAUDE.md 中定义的 slash commands 让 AI Agent 理解可用的操作。这对纯 AI-first 工具设计非常有启发性。

3. **个人数据即基础设施**：
   `documents/` 文件夹被设计为"个人数据仓库"，/setup 自动检测已有材料 vs 引导填写——这是一种增量构建个人数据的优雅模式。

### 同类需求的可参考思路

如果有人想做"AI 简历优化器/求职助手"，这个仓库提供了完整的参考架构：**画像管理 → 岗位搜索 → 匹配评估 → 文档生成 → 质量审计** 的五步工作流。其中的自画像模板（CLAUDE.md 的结构化 Profile）是最值得复用的资产。

## 🧠 核心源码解读

### 1. CLAUDE.md——Agent 入口层（核心设计）

CLAUDE.md 定义了 4 个关键 slash command，构成了整个框架的"API"：

```
/setup    → 三路径画像：自动扫描 documents/ 文件夹 / 粘贴整份简历 / 访谈式填写
/scrape   → 多平台搜索 → 去重 → 按匹配度排序
/rank     → 批量评分 → 排名短名单
/apply    → 评估匹配 → 定制简历 → 撰写求职信 → 审查修改 → 编译 PDF
```

**设计亮点**：所有命令的语义通过 CLAUDE.md 中的自然语言描述和工作流说明定义，而不是通过代码。这意味着**修改工作流 = 修改 CLAUDE.md，不需要改代码**。这是 AI Agent 时代的新架构模式。

### 2. 匹配评估框架（隐藏在 CLAUDE.md 中的决策逻辑）

框架内置了一个多维度的匹配评估标准，而不是简单的关键词扫描：

```
评估维度:
- 核心技能匹配度（硬性门槛）
- 经验相关性（行业/角色/规模）
- 行为特质匹配（文化适配）
- 成长空间（挑战 vs 舒适区）
- 薪资基准（可选，基于公开数据）
```

这里的"薪资基准"功能——基于公开数据集估算各职位薪资水平——是 README 没有专门强调但实际非常有价值的功能。

### 3. ATS 可解析性检查

```python
# /apply 命令的 ATS 审计步骤（简化语义）：
# 1. lualatex 编译 PDF
# 2. pdftotext 提取文本
# 3. 与 JD 关键词交叉匹配
# 4. 报告匹配率 + 改进建议
```

如果 `pdftotext` 不可用，自动降级为视觉关键词审查——这是实践中常见的**优雅降级**模式。

## 📐 架构决策与设计哲学

- **Replit 可运行性**：项目支持直接在 Replit/Glitch/Railway 上运行，核心代码无需本地安装
- **不要求全自动化**：用户仍需要选择哪些岗位值得投递，Agent 只是加速而非替代决策——这种"人机协作"定位比完全自动化的方案更务实
- **丹麦市场优先**：搜索适配器集中在丹麦平台，但架构上预留了替换接口——这是 MVP 优先的务实策略

### 差评共识&踩坑高发区

- **LaTeX 编译坑**：`pdflatex` 在高版本的 MiKTeX 上会因 `fontawesome5` 字体扩展问题而失败，必须用 `lualatex`——这是新手最容易碰壁的地方
- **仅支持丹麦招聘平台**：搜索适配器主要面向丹麦市场（Jobindex, Jobnet, Akademikernes Jobbank），其他国家需要自行编写搜索 Skill
- **对 Claude Code 的强依赖**：框架完全绑定 Claude Code，目前不支持其他 AI 编码助手（如 Codex、Cursor）

### 争议焦点

- "这是 AI 求职自助还是 AI 生成垃圾简历？"——部分招聘者担心 AI 生成的求职信缺乏个性化。但作者的设计哲学（"我的目标是做真实的自己，在最好的光线下呈现"）对此给出了回应：AI 不虚构不存在的技能

## 🎯 核心研判

### 项目优势（不可替代的价值点）

1. **"非程序员 Agent 工程"标杆**：证明了 AI Agent 确实降低了软件开发门槛，对"AI 会让什么人更有能力"这个命题给出了实证
2. **Drafter-Reviewer 双 Agent 质量闭环**：在所有求职工具中独树一帜，也是最值得其他 Agent 应用借鉴的设计模式
3. **CLAUDE.md 即 API 的架构范式**：对 AI-first 应用设计非常有启发性

### 项目风险

1. **Claude Code 单平台绑定**：如果 Claude Code 变更其 CLAUDE.md 规范，项目需要相应适配
2. **招聘平台反爬**：搜索适配器依赖网页爬取，平台可能随时添加反爬措施
3. **维护者非职业开源**：作者是地球物理学家，无法保证持续的高频更新

### 适用场景 & 不适用场景

✅ **适合**：Claude Code 重度用户、需要批量精准求职的技术从业者、丹麦求职市场
❌ **不适合**：非 Claude Code 用户、对 LaTeX 完全不熟悉的求职者、非丹麦市场的求职者（需自行写搜索适配器）

### 趋势判断

📈 **上升期**：AI 求职工具正处于市场上升周期，但项目本身热度已从爆发期（3 月数十万观看）进入稳定增长期。其架构设计（双 Agent 审查）是行业级思路，但具体实现较绑定生态。
