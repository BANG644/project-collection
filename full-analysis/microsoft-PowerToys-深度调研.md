- **MIT 许可证 + 商业友好**：微软选择最宽松的开源许可，鼓励第三方集成

## 🧠 核心源码解读

PowerToys 的架构核心是**模块管理器（runner）** 和**各独立模块**：

### 模块发现机制

各模块通过注册表/配置文件声明自己的能力，runner 在启动时枚举所有可用模块。这种松耦合设计使得：

- 用户可以仅启用自己需要的工具
- 新模块可以独立开发、独立测试
- 微软团队和社区贡献者可以各自维护不同模块

### FancyZones（最受欢迎的工具）

FancyZones 是 PowerToys 中下载量最高、口碑最好的工具。核心设计：

- 编辑器模式：用户定义窗口布局模板（网格、行、列、自定义）
- 运行时：通过 Win32 API 钩子拦截窗口移动事件，自动吸附到最近的分区
- 快捷键：Win+` 快速在不同布局间切换

### 与 Windows 的集成深度

PowerToys 的独特优势在于它能调用 Windows 内部 API。例如：

- Keyboard Manager 通过底层键盘钩子实现按键重映射
- File Explorer Add-ons 直接注入文件资源管理器的预览面板
- Peek 类似 macOS 的 Quick Look，通过 Shell 扩展实现

### 差评共识 & 踩坑高发区

- **版本兼容问题**：某些版本会出现崩溃，尤其是预览版
- **性能开销**：开机启动后内存占用 200-400MB，部分用户认为偏大
- **学习曲线**：30+ 工具太多，新用户不知道从哪个开始
- **部分工具不稳定**：Workspaces 等新功能仍有 Bug

### 争议焦点

- **Workspaces 功能**：2025 年引入的"保存和恢复窗口布局"功能，被部分用户认为是 FancyZones 的重复功能
- **是否应该预装进 Windows**：社区长期讨论"微软为什么不直接把 PowerToys 内置进 Windows"

**选择建议**：
- **日常效率提升** → PowerToys（一个安装包解决问题）
- **需要自动化脚本** → PowerToys + AutoHotkey 组合
- **仅截图需求** → Snipaste（更专注、更轻量）
- **仅多显示器管理** → DisplayFusion（更专业）

## 🎯 核心研判

### 项目优势

- **135K ⭐ 和 20+ 万日活用户**证明了项目的真实价值和持续生命力
- MIT 开源 + 微软官方维护 = 长期可用性和商业友好的平衡
- v0.100 里程碑标志着项目已跨越"实验性"阶段

### 项目风险

- 项目体量巨大（C#/C++/TS 混合），贡献门槛高
- 部分工具功能与 Windows 内置功能重叠（如截图工具 vs Win+Shift+S）
- 性能开销限制了低配设备上的使用体验

### 适用场景 & 不适用场景

**适用**：Windows 10/11 用户，尤其是需要多窗口管理、批量文件操作、快捷键定制的开发者/设计师/办公人员

**不适用**：低配置设备、Linux/macOS 用户、仅需要单一功能（可用更轻量的替代品）

### 趋势判断

**稳定上升期**。v0.100 发布后，PowerToys 已从"实验项目"进入"成熟工具"阶段。随着 Windows 11 的普及，其窗口管理功能的需求还会增长。AI 功能的集成（如 Advanced Paste 的 AI 粘贴）是未来的重要增长方向。
