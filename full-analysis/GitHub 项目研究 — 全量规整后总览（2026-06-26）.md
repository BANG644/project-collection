## 🔍 调研方法论（6 维度深度调研）

每份报告遵循 `github-repo-deep-dive` 6 维度框架：

1. **项目定位** — 一句话定位 + 解决什么问题
2. **项目全景** — Stars/Forks/语言/许可/活跃度等基础档案
3. **核心架构** — 目录结构 + 关键模块 + 架构决策
4. **源码深度解读** — 核心文件走读 + 设计模式识别
5. **全网口碑** — Issue/PR/社区评价交叉验证
6. **竞品对比 + 核心研判** — 横向对比 + 优势/风险/入场时机判断

**铁律**：如果调研报告内容用户看 README 也能得到，那就是失败的调研。
