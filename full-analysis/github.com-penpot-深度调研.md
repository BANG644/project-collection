## 一、项目定位

Penpot 是**全球首个面向跨职能团队的开源设计与原型平台**，对标 Figma，但完全开源、支持 Docker 自部署。核心理念是：

- **设计表达为代码**（SVG、CSS、HTML、JSON）
- **开放标准 + 私有化部署**，无供应商锁定
- **设计与开发协作**，而非设计完成后"扔过墙"
- **MCP 服务**打通设计→代码的双向流程

2026 年 Penpot 已被 Digital Public Goods Alliance 认定为数字公共产品。

## 四、核心创新亮点

### 1. MCP Server（2025-2026 重大更新）

Penpot 2026 年最大的亮点是原生 MCP 服务器，支持：

- AI 工具直接读取/编辑 Penpot 设计文件
- 多方向设计⇄代码工作流（非传统"设计→开发"单向）
- 与 Claude / Cursor / Codex 等编程 Agent 集成
- 设计系统中的 AI 驱动自动化

### 2. Design Tokens（单源真相）

原生设计令牌系统解决了一个长期痛点——设计与开发用同一套命名/值体系维护 UI 属性（颜色、间距、字体）。令牌变更时设计和代码同步更新。

### 3. CSS Grid + Flex 布局

Penpot 是目前唯一原生支持 CSS Grid 定义的面包板工具。设计出来的布局直接对应 CSS 代码，减少"设计很难转代码"的鸿沟。

### 4. SVG 原生 + WASM 渲染

底层使用 SVG 作为原生格式（非 Figma 的自有格式），前端渲染通过 WASM 加速，使得大文件的渲染性能接近原生工具。

## 八、核心研判

- **优势**：完整开源（MPL 2.0）+ SVG 原生标准 + 可自部署，无供应商锁定；MCP Server 让它从"设计工具"升级为"AI 设计⇄代码双向协作枢纽"；Design Tokens + CSS Grid 原生支持直击"设计难转代码"痛点；已被认定为数字公共产品（DPGA）。
- **风险**：前端基于 ClojureScript，二次开发/插件生态的入门门槛高于 React/TS 生态的 Figma；与 Figma 的实时协作流畅度、企业 Admin 成熟度仍有差距；MCP/AI 能力虽领先开源界，但闭源 Figma AI 在模型资源上更激进。
- **趋势**：设计工具正在分化为"闭源 SaaS（Figma）"与"开源自部署（Penpot）"两条路线；MCP 让设计系统成为 Agent 可直读/直写的"活文档"，是 AI-native 产品团队的必然选择。
- **启发**：给 AI 产品/工具构建者的启示——把"设计系统"做成机器可读的单源真相（Design Tokens + MCP），你的产品就能被编程 Agent 直接消费；开源 + 自部署在合规敏感行业（金融/政务/医疗）是不可替代的护城河。
